# NAMCore oracle

The parity oracle for A2 (and a cross-check for A1). It is NeuralAmpModelerCore
v0.5.3 itself, pinned as a recursive submodule, wrapped in a tiny C++ harness.
Fixtures generated by it are committed; you only need this to *regenerate* them.

## Build (one-time)

    git submodule update --init --recursive      # if you didn't clone --recursive
    ./tests/oracle/build.sh                       # -> tests/oracle/build/oracle

Requires CMake >= 3.10 and a C++20 compiler. The generic WaveNet path is used
(the a2_fast specialization is intentionally off) so the oracle matches the
engine nam-rs ports.

## Use

    # (model.nam, input.json) -> output.json
    python3 tests/oracle/oracle_gen.py <model.nam> <input.json> <output.json>

The harness runs a fresh `Reset` with **no prewarm** — zero-init streaming,
matching nam-rs. Compare steady-state (skip the receptive field), as
`tests/parity.rs` does.

## Validate the harness

    python3 tests/oracle/check_a1_crosscheck.py   # NAMCore vs torch A1 fixture, 1e-5

## Updating the pin

Check out a new tag in the submodule, re-run the cross-check, and regenerate all
oracle-derived fixtures.
